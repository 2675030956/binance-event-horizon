# 产品逻辑

## 产品定位

`币安事件地平线` 不是行情终端，而是“未来情景引擎”。

它试图解决的不是“现在谁涨了”，而是：

- 哪些信号正在共振
- 哪些叙事正在被官方催化
- 哪些资产已经进入高杠杆 / 高拥挤区
- 接下来 6h / 24h 最值得盯住的路径是什么

## 六层结构

### 1. `event_horizon`

全局相位层，回答三个问题：

- 市场现在偏点火、分歧、扩张还是压缩
- 多源信号密度高不高
- 风险和机会的重心偏向哪边

### 2. `signal_constellation`

信号星图层，不按“接口 A / B / C”展示，而按更容易讲故事和演示的轨道层展示：

- AI 叙事束
- Alpha 前沿层
- 聪明钱跃迁层
- 杠杆热层
- 官方催化带
- Launchpad / 迷因冲击层

### 3. `scenario_engine`

把多源数据转成未来情景卡。

每张卡必须包含：

- 场景标题
- 情景类型
- 时间窗口
- 触发条件
- 失效条件
- 当前为何成立

### 4. `catalyst_reactor`

把官方公告和产品动作映射成“催化路径”。

例如：

- 上新 -> 现货承接 -> 合约扩散
- Margin / Earn / Convert 接入 -> 流动性再定价
- API / 产品更新 -> 交易系统侧行为变化

### 5. `gravity_alerts`

这个层故意和机会层分开，防止“高热度 = 好机会”的错觉。

典型预警来自：

- 高资金费率
- 持仓 5 分钟异常跃迁
- 审计风险 / 敏感标签
- 热度大于流动性
- DEV / Insider / Wash Trading 信号

### 6. `orbit_watchlist`

这是给 OpenClaw、交易助理、内容团队或人工盯盘直接复用的执行接口。

每个轨道卡都要有：

- 当前优先级
- 观察逻辑
- 触发条件
- 失效条件
- 时间窗口

## 评分框架

每个资产会被拆成 7 个维度：

- `market_score`：价格动能 + 成交额 + 多源覆盖
- `social_score`：社媒热度 + 情绪
- `smart_money_score`：聪明钱流入 / 智能信号
- `official_score`：公告与官方动作相关性
- `leverage_score`：资金费率 + 持仓变化 + 合约强度
- `alpha_score`：Alpha / 4x points / 早期交易强度
- `risk_score`：审计风险 + 低流动性 + 可疑标签 + 拥挤风险

最终不会只给一个总分，而是保留拆分项，方便其他 OpenClaw 继续接力。

## 市场相位

### `点火窗口`

催化压力高，机会分数领先，风险还没有全面失控。

### `奇点扩张`

市场在高信号密度、高杠杆、高关注状态下共振。

### `高热分歧`

社媒和价格很热，但资金与质量确认并不完全一致。

### `引力压缩`

机会不足、催化不强，或者风险层显著抬升。

## 与其它方案的区别

这套项目故意与以下风格区分：

- 不是 Alpha 的量化刷分面板
- 不是 Narrative OS 的编辑部工作台
- 不是风控待检清单

它更像一个“赛博驾驶舱 + 未来情景推演台”。
